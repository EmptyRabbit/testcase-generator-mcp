## 目标

你需要将用户提供的测试用例 markdown 文档按下列规则精确转换为 JSON 格式的思维导图数据，再调用 `generate-mind-map`
工具（xmind-generator-mcp）将该 JSON 与文件名一起传入，生成 .xmind 文件

## 上下文信息

用户在对话中提供以下信息：

- Markdown 格式
  ```markdown
  # 需求名

  ## 一级模块名

  ### 二级模块名

  #### 三级模块名（可选）

  ##### 四级模块名（可选）   
  - 【P0】【用例ID】测试用例描述 # 无序列表，标明优先级
    - 步骤1：操作步骤描述 # 子项，缩进4空格
    - 步骤2：操作步骤描述
      - 预期结果1：验证内容的描述
      - 预期结果2：验证内容的描述
    - 步骤3：操作步骤描述
      - 预期结果3：验证内容描述 # 只有1个预期结果，也需要有编号
  ```
- Markdown 格式说明
  ```markdown
  - 使用多个 `#` 表示模块层级，最多5层，即最多为 `#####`，其中第 1 层为需求名，第 2–5 层对应一级到四级模块
  - 用例描述前标注优先级：`【P0】`、`【P1】`、`【P2】`、`【P3】`
  - 用例ID为 TC+4位数字，例如：TC0001
  - 步骤和结果交替排列，逻辑清晰
  - 子项使用4个空格缩进
  ```

## 核心任务

所有子任务：分批生成 JSON，完整性自检与合并，生成 .xmind 文件

**重要：你的任务是必须严格按照以下步骤顺序执行子任务，不得跳过任何步骤。**

**执行禁忌：不得运行任何脚本/代码做 Markdown→JSON 转换（含仓库内已有脚本）。**

1. 第一步（必须执行）：分批生成 JSON
2. 第二步（必须执行）：完整性自检与合并
3. 第三步（必须执行）：生成 .xmind 文件

## 子任务内容

### 1. 分批生成 JSON

- 步骤1：了解下方附录中的 `转换映射规则`、`输出JSON结构` 和 `处理规则` 的内容
- 步骤2：如果 Markdown 内容较长，按一级模块拆分，根据附录中的内容分批转换成 JSON；不得通过运行脚本/代码生成
- 步骤3：确保每个 JSON 片段格式正确
- 步骤4：此任务执行过程中不允许调用 `generate-mind-map` 工具

### 2. 完整性自检与合并

- 步骤1：将所有分批生成的 JSON 片段合并为一个完整的 JSON 对象（含 `title` 和完整的 `topics` 数组）
- 步骤2：检查合并后 JSON 数据的最后一个节点（Last Topic）
- 步骤3：对比 Markdown 文档的最后一部分内容
- 步骤4：如果两者不一致，说明生成被截断或遗漏，必须重新生成缺失的部分并合并
- 步骤5：此任务执行过程中不允许调用 `generate-mind-map` 工具

### 3. 生成 .xmind 文件

- 步骤1：调用 `generate-mind-map` 工具，传入该完整 JSON 的 `title`、`topics` 以及 `filename`（.xmind 文件名），生成 .xmind 文件

## 附录

### 转换映射规则

| Markdown元素 | JSON位置 | 说明 |
| :--- | :--- | :--- |
| `# 需求名`（一级标题） | 根节点的 `title` | 整图的根标题，与需求名一致 |
| `##`～`#####` 标题 | 按层级进入 `topics[].children[]` | 仅指标题行，共 5 级，对应各级主题 |
| `- 【Px】【用例ID】描述` | 对应层级的`children[].title` | 用例项，保留优先级与用例ID |
| `- 步骤n：...` | 该用例的`children[]` | 步骤节点，n 为数字 |
| `- 预期结果n：...` | 紧邻上一步骤的`children[]` | 预期结果挂在对应步骤下，n 为数字 |
| 缩进子项 | 上级节点的`children[]` | 根据缩进层级确定父子关系 |

### 输出JSON结构

```json
{
  "title": "需求名",
  "topics": [
    {
      "title": "一级模块名",
      "children": [
        {
          "title": "二级模块名",
          "children": [
            {
              "title": "【P0】【TC0001】测试用例描述",
              "children": [
                {
                  "title": "步骤1：操作步骤描述"
                },
                {
                  "title": "步骤2：操作步骤描述",
                  "children": [
                    {
                      "title": "预期结果1：验证内容的描述"
                    },
                    {
                      "title": "预期结果2：验证内容的描述"
                    }
                  ]
                },
                {
                  "title": "步骤3：操作步骤描述",
                  "children": [
                    {
                      "title": "预期结果3：验证内容描述"
                    }
                  ]
                }
              ]
            }
          ]
        }
      ]
    }
  ]
}
```

### 处理规则

1. **文本处理：** 去除 markdown 标记符（#、-），保留完整文本内容
2. **标题层级：** 仅指标题行，支持 `#`～`#####` 共 5 级标题，按层级映射到主题树；`# 需求名` 作为根 `title`，`##` 及以下进入`topics` 树。
3. **层级处理：** 根据缩进层级（支持 2 或 4 空格）识别父子关系，构建 JSON 嵌套结构；务必确保"步骤"节点挂在"用例"下，"预期结果"节点挂在"步骤"下。
4. **空节点：** 没有子项的节点不包含 `children` 字段（也不要写 `"children": []`）。
5. **特殊字符：** 保留优先级标识【P0】～【P3】及用例ID（如【TC0001】）的原始格式
6. **编码：** 确保中文字符正确处理
